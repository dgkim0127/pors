# 보안/운영 체크리스트

## 현재 적용된 보호

- 관리자키 로그인은 12시간 뒤 만료됩니다.
- 관리자키를 5회 틀리면 5분 동안 재시도가 제한됩니다.
- 카테고리, 품목, 거래처 입력값은 길이와 숫자 범위를 제한합니다.
- 판매 내역은 로컬 저장소가 과도하게 커지지 않도록 최근 500건까지만 보관합니다.
- 카테고리 삭제 전 확인창을 띄우고, 삭제 시 해당 품목도 같이 제거합니다.
- 미리보기 중 서비스워커 캐시를 끄고 기존 등록을 해제해 오래된 화면이 남는 문제를 줄였습니다.
- Supabase SQL은 익명 사용자가 모든 테이블을 수정하는 임시 정책을 제거했습니다.
- 기본 CSP를 넣어 스크립트, 연결, 이미지 출처를 제한했습니다.

## APK 배포 전에 꼭 할 일

- `app-config.js`와 `public/app-config.js`의 `PIERCE_ADMIN_KEY`를 `0000`이 아닌 매장 전용 키로 바꿉니다.
- 최종 APK 빌드에서는 React CDN 의존을 없애고 `npm install && npm run build`로 앱 자산을 APK 내부에 포함합니다.
- Supabase를 실제로 사용할 경우, `supabase/schema.sql`의 안전 기본값을 유지한 뒤 매장별 `store_id`가 검증되는 RPC/RLS 정책을 별도로 추가합니다.
- APK를 공유할 때는 테스트용 APK와 운영용 APK를 구분하고, 운영용 키가 들어간 파일을 공개 저장소에 올리지 않습니다.

## 남은 보안 한계

- 클라이언트에 들어간 관리자키는 APK를 분석하면 노출될 수 있습니다. 중요한 매출 데이터를 여러 기기에서 공유하려면 서버/Supabase RPC에서 관리자키를 검증해야 합니다.
- 로컬 저장소 데이터는 태블릿을 직접 가진 사람에게 완전히 숨길 수 없습니다. 태블릿 잠금, 백업 정책, 직원 사용 규칙이 필요합니다.
